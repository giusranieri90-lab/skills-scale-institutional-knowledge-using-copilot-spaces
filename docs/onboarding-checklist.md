# Onboarding Checklist

Checklist for the Onboarding Coordinator to ensure new contributors ramp quickly and have the required access and context.

## Before start date
- [ ] Provision accounts and access (GitHub, CI, tracking tools)
- [ ] Share team handbook, process docs, and reading list
- [ ] Schedule onboarding meetings with key contacts (PM, Tech Lead, Buddy)

## First week
- [ ] Run orientation session covering processes and expectations
- [ ] Ensure the new contributor can run the project locally and run tests
- [ ] Assign a buddy for day-to-day questions

## First month
- [ ] Confirm training completion and necessary compliance/security trainings
- [ ] Review progress and adjust onboarding materials if gaps are found
- [ ] Collect feedback to improve the onboarding experience
